安装Grunt

npm uninstall -g grunt

grunt从版本0.3.X到0.4.x，变化比较大,如果之前安装了0.3.x版本，需先进行卸载

npm install -g grunt-cli

grunt-cli的主要作用是让我们可以运行Grunt命令，加上-g，则可以在任意目录下运行

Grunt配置文件

1.package.json

用于保存项目元数据,运行 npm install 就会为文件中所列出的每个依赖安装对应的版本。

2.Gruntfile.js

用于配置或定义任务以及加载 Grunt 插件。

package.json--范例

{

"name": "GruntJs",

"version": "0.0.0",

"description": "GruntJs",

"main": "index.js",

"keywords": ["GruntJs"],

"author": "wkylin",

"license": "ISC"

}

package.json规范

https://www.npmjs.org/doc/json.html

Gruntfile配置文件：

1.包装函数

2.项目与任务的配置

3.加载 Grunt 插件和任务

4.默认/自定义任务

1.包装函数：

module.exports = function(grunt) {

// 处理 Grunt

};

2.项目与任务的配置：

grunt.initConfig({

pkg: grunt.file.readJSON('package.json'),

uglify: {

//任务配置

}

});

3.加载 Grunt 插件和任务：

// 加载提供 "uglify" 任务的插件

grunt.loadNpmTasks('grunt-contrib-uglify');

4.默认/自定义任务：

//default tasks

grunt.registerTask('default', ['uglify']);

//user-defined tasks

grunt.registerTask('buildcss', ['cssmin']);
